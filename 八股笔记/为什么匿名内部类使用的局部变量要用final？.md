因为**匿名内部类会“捕获”它所在方法里的局部变量**，而这个局部变量本身是存放在**栈帧**里的，方法执行结束后就没了；但匿名内部类创建出来的对象**可能还会继续存活**。  
所以，匿名内部类实际上拿到的不是这个局部变量本身，而是**它的一个拷贝**。

一旦允许这个局部变量后续继续修改，就会出现歧义：  
你改的是外部方法里的那个变量，还是匿名内部类里保存的那个拷贝？

为了避免这种问题，Java 就要求：

- 早期版本：必须显式写 `final`
    
- Java 8 开始：可以不写 `final`，但必须是 **effectively final（事实上不再修改）**


### 为什么 Java 8 后又不用显式写 final 了？

因为编译器会自动帮你判断。只要这个变量赋值后**没有再被修改**，它就是 effectively final，照样可以被匿名内部类或 lambda 使用。
